# UniFi OS Server Port 8443 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move the `gk` UniFi OS Server host HTTPS listener from TCP 11443 to TCP 8443.

**Architecture:** Keep the reusable module and container's internal TCP 443 unchanged. Override only the `gk` host's `webPort`, build and activate the resulting NixOS generation, then verify both positive and negative listener behavior before merging.

**Tech Stack:** NixOS, Nix flakes, systemd, rootless Podman, SSH.

## Global Constraints

- Work only on `codex/unifi-port-8443` until live acceptance passes.
- Preserve all existing UniFi volumes and rollback backups.
- Do not expose MongoDB on the host.
- Do not push the repository.

---

### Task 1: Declare the gk host port

**Files:**
- Modify: `systems/x86_64-linux/gk/configuration/unifi.nix`
- Test: `tests/unifi-os-server-vm.nix` (run unchanged)

**Interfaces:**
- Consumes: `services.unifi-os-server.webPort`, which maps the selected host TCP port to container TCP 443.
- Produces: a `gk` system closure publishing UniFi HTTPS on host TCP 8443.

- [ ] **Step 1: Record the baseline**

Run:

```bash
nix eval --json .#nixosConfigurations.gk.config.services.unifi-os-server.webPort
```

Expected: `11443`.

- [ ] **Step 2: Change the host override**

Set the existing option to:

```nix
webPort = 8443;
```

- [ ] **Step 3: Verify the declared value**

Run the Step 1 command again. Expected: `8443`.

- [ ] **Step 4: Run repository verification**

Run:

```bash
git diff --check
nix build .#nixosConfigurations.gk.config.system.build.toplevel --no-link -L
nix build --impure --expr 'let flake = builtins.getFlake (toString ./.); pkgs = import flake.inputs.nixpkgs { system = "x86_64-linux"; }; in import ./tests/unifi-os-server-vm.nix { inherit pkgs flake; }' --no-link -L
```

Expected: all commands exit zero.

- [ ] **Step 5: Commit the declarative change**

```bash
git add docs/superpowers/specs/2026-07-14-unifi-port-8443-design.md docs/superpowers/plans/2026-07-14-unifi-port-8443.md systems/x86_64-linux/gk/configuration/unifi.nix
git commit -m "Move gk UniFi HTTPS to port 8443"
```

### Task 2: Activate and accept the live port

**Files:**
- Modify: none
- Test: live `gk` system and controller endpoints

**Interfaces:**
- Consumes: the built NixOS `gk` system closure from Task 1.
- Produces: an accepted live generation or a restored prior generation.

- [ ] **Step 1: Confirm rollback state and copy the closure**

Record `readlink -f /run/current-system` and the current system generation on
`gk`, then use the repository's existing deployment workflow to activate the
new closure.

- [ ] **Step 2: Verify the service and listeners**

On `gk`, require `unifi-os-server.service` to be active with a successful
result, TCP 8443 to listen, and TCP 11443 and TCP 27017 not to listen.

- [ ] **Step 3: Verify controller behavior**

Require HTTPS on `https://192.168.1.3:8443/` to respond and all three restored
devices to remain present and reachable.

- [ ] **Step 4: Roll back on any failed acceptance check**

Activate the recorded prior NixOS generation, then require the service and
`https://192.168.1.3:11443/` to recover before reporting the failure.

- [ ] **Step 5: Merge accepted work**

Fast-forward `codex/unifi-port-8443` into `master`, run the final `gk` build
from `master`, and leave the remote repository untouched.
